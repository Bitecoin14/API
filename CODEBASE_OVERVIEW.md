# Hand Tracker — Codebase Overview

> Last updated: 2026-04-29

---

## What This Project Does

Real-time computer vision application that runs entirely on a local laptop webcam. It combines:

- **Hand skeleton tracking** via MediaPipe (21 landmarks, 2 hands)
- **Body pose detection** with 10 body-gesture classifiers
- **Face recognition** using a 3-model ensemble (InsightFace + AdaFace + ElasticFace)
- **8 visual filters** cycled by a pinch gesture
- **ASL fingerspelling** recognition (A–Z, rule-based + optional RandomForest)

All processing is local. No cloud APIs, no internet required at runtime.

## Architecture

### Three-Thread Pipeline

```
[Camera]
   │
   ▼
CaptureStage ──→ raw_queue (maxsize=2)
                    │
                    ▼
             InferenceStage ──→ result_queue (maxsize=2)
                                    │
                                    ▼
                              Render stages (main thread):
                              FilterStage → FaceStage → ASLStage → RendererStage
                                    │
                                    ▼
                              cv2.imshow (screen)
```

Queues provide natural backpressure: if the render thread falls behind, the capture thread stalls rather than building up a backlog.

### Stage Protocol

Each stage implements:
```python
def process(ctx: FrameContext) -> FrameContext: ...
def close() -> None: ...
```

Stages are stateful but isolated — they communicate only through `FrameContext`.

### FrameContext Fields

| Field | Set by | Read by |
|-------|--------|---------|
| `frame` | CaptureStage | all stages (mutable) |
| `raw_frame` | CaptureStage | FilterStage (preserve original) |
| `hand_results` | InferenceStage | FilterStage, ASLStage, RendererStage |
| `pose_results` | InferenceStage | RendererStage |
| `face_results` | FaceStage | RendererStage |
| `asl_letters` | ASLStage | RendererStage |
| `capture_fps` | CaptureStage | RendererStage |
| `config` | Pipeline constructor | all stages |

## Face Recognition Pipeline

```
Frame
  │
  ▼
DualDetector
  ├── RetinaFace (InsightFace buffalo_l) — primary, 640×640
  └── YOLOv8-face (ultralytics) — cross-check, 320×320
  → Only faces with IoU ≥ 0.5 between both detectors pass
  │
  ▼
Quality Gate
  ├── Face width ≥ 80px
  ├── Detection score ≥ 0.65
  ├── Yaw ≤ 35°
  └── Blur (Laplacian variance) ≥ 50
  │
  ▼
EnsembleRecognizer (3 independent models)
  ├── Model A: InsightFace antelopev2 (ArcFace, ResNet100, Glint360K)
  ├── Model B: AdaFace IR-ResNet101 (WebFace12M) — quality specialist
  └── Model C: ElasticFace-Arc+ (ResNet100, MS1MV3) — robustness specialist
  → Each returns (best_match, best_score, runner_up, ru_score)
  │
  ▼
Arbitrator (5 decision rules)
  ├── CONFIRMED: all models agree, all scores above threshold
  ├── SOFT_CONFIRMED: 2/3 agree, both above threshold
  ├── LOW_CONFIDENCE: unanimous but weak scores, clear margin
  ├── AMBIGUOUS: models disagree or margins too thin
  └── UNKNOWN: all scores below threshold
  │
  ▼
TemporalSmoother (IoU-based tracker)
  ├── 15-frame rolling window per track
  ├── 60% consensus required before displaying a name
  └── 3-frame initialization delay (prevents flash-guesses)
  │
  ▼
RendererStage
  └── Color-coded bounding box + name label
      ├── Green: CONFIRMED
      ├── Cyan-yellow: SOFT_CONFIRMED
      ├── Orange: LOW_CONFIDENCE
      ├── Red: AMBIGUOUS
      └── Grey: UNKNOWN
```

### CPU Fallback

On CPU (no CUDA), Model C (ElasticFace) is skipped automatically when `CPU_FALLBACK_TWO_MODELS = True`. The system runs 2-model consensus (A + B). This keeps inference fast enough for real-time use.

### Graceful Degradation

If all InsightFace/ONNX dependencies are missing, `FaceStage` falls back to the legacy OpenCV LBPH recognizer using the `known_faces/` directory.

## Operating Modes

| Mode | Command | Purpose |
|------|---------|---------|
| `hand` | `python hand_tracker.py` | Hand skeleton + filters (default) |
| `face` | `python hand_tracker.py --mode face` | Face recognition only |
| `enroll` | `python hand_tracker.py --mode enroll` | Register new people (webcam) |
| `enroll --from-folder` | `python hand_tracker.py --mode enroll --from-folder ./photos/` | Batch enroll from images |
| `audit` | `python hand_tracker.py --mode audit` | Confusable-pairs report |

Optional feature flags (additive, work in hand mode):

| Flag | What it adds |
|------|-------------|
| `--face` | Face recognition overlay |
| `--pose` | Body pose skeleton + gesture panel |
| `--blur` | Middle-finger blur effect |
| `--asl` | ASL fingerspelling overlay |

## Key Configuration Constants (`core/config.py`)

| Constant | Value | Meaning |
|----------|-------|---------|
| `FACE_MIN_SIZE` | 80px | Minimum face width to process |
| `FACE_MIN_DET_SCORE` | 0.65 | Minimum detection confidence |
| `FACE_MAX_YAW` | 35° | Maximum face turn before skipping |
| `FACE_MIN_BLUR` | 50 | Minimum Laplacian variance (sharpness) |
| `MODEL_A_THRESHOLD` | 0.45 | Cosine similarity floor for Model A |
| `MODEL_B_THRESHOLD` | 0.42 | Cosine similarity floor for Model B |
| `MODEL_C_THRESHOLD` | 0.43 | Cosine similarity floor for Model C |
| `AMBIGUOUS_MARGIN` | 0.07 | Required gap between top-2 matches |
| `TRACK_WINDOW_SIZE` | 15 | Frames in temporal smoothing window |
| `TRACK_MIN_CONSENSUS` | 0.60 | Fraction needed to display a name |
| `TRACK_INIT_FRAMES` | 3 | Identical decisions before first display |
| `CONFUSABLE_THRESHOLD` | 0.35 | Similarity above which pairs are warned |
| `CPU_FALLBACK_TWO_MODELS` | True | Skip Model C on CPU |

## Dependencies

| Package | Purpose | Required |
|---------|---------|---------|
| `opencv-contrib-python` | Camera, drawing, Haar cascade | Always |
| `mediapipe` | Hand + Pose landmarks | Always |
| `numpy` | Embedding math, array ops | Always |
| `scikit-learn` | ASL RandomForest classifier | For ASL model |
| `joblib` | Model serialization | For ASL model |
| `insightface` | RetinaFace detector + antelopev2 embedder | Face recognition |
| `onnxruntime-gpu` | AdaFace + ElasticFace inference | Face recognition |
| `ultralytics` | YOLOv8-face cross-checker | Face recognition |

MediaPipe is pinned to `<=0.10.14` because 0.10.15+ removed the legacy `solutions` API used by this project.
